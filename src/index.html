Theme System

The theme system includes base color values, typography sizes, and handful of other reusable utility variables.

Overriding Theme Variables

The theme variables listed in each section, such as $d-color-background, can easily be overridden. Simply redefine the variable with your new value prior the import of the theme file, which is the first import in the main style.scss file since the variables are used in the other imports.

You can of course modify the theme file directly, but overrides are handy for testing the theme out or if you plan to reuse this template for multiple projects, or across multiple page templates.

Colors

Color Function

You can access theme colors within your custom styles using the d-color() function which takes the name of any colors and returns the defined value. This makes the colors extensible without copying color codes directly.

Color Options and Values

  • #171392 $d-color-primary .d-[ink|background]--primary
  • #df2b7c $d-color-secondary .d-[ink|background]--secondary
  • #f9f9f9 $d-color-background .d-background
  • * #252450 $d-color-body .d-ink--text
  • * #5e5d6b $d-color-gray .d-[ink|background]--gray
  • #fff $d-color-light .d-[ink|background]--light
  • rgba(black, 0.87) $d-color-dark .d-ink--dark
  • Link default: $d-link-color: d-color("primary")

* These values are computed from the $d-color-primary value unless overridden

Color Maps

Maps are provided to selectively generate `ink` and `background` colors to allow you to reduce total generated classes if you aren't using some of them.

Add/remove to selectively generate `ink` (text color) classes:
$d-ink-colors: "primary", "secondary", "text", "gray", "light", "dark";

Add/remove to selectively generate `background` classes:
$d-background-colors: "primary", "secondary", "background", "gray", "light";

Typography

Update the font family by removing or changing the Google font stylesheet link in the index, and then update the font stack on the $d-font-family variable. The included font is Baloo 2.

  • Headline h1 $d-h1-font-size h1 | .d-h1
  • Headline h2 $d-h2-font-size h2 | .d-h2
  • Headline h3 $d-h3-font-size h3 | .d-h3
  • Headline h4 $d-h4-font-size h4 | .d-h4
  • lead - utility $d-lead-font-size .d-lead
  • text $d-text-font-size p | li
  • Text alignment: .d-text-align--[left|center|right]

Components

Buttons

All buttons require the base .d-button class, which by default is a filled-in button using the primary color as background.

Add/remove to selectively generate button colors variants:
$d-button-variants: "primary", "secondary", "light", "gray";

Apply a color with a class structured as .d-button--[color] where `color` is one of the available variants from the map.

default secondary light gray

A second variant of .d-button-outlined is available.

default secondary light gray

Reduce button size with .d-button--small. Add icons (svg preferred) inside span.d-button__icon, with added .d-button__icon--end if it follows the button text.

Layout

Inherited Element Styles

Minimal element styles are placed on the following:

  • body: notably - display: flex; flex-direction: column;
  • main: vertical padding
  • article: max-width: 80ch;
  • typography elements: h1-h4, p, ul, li, code, pre, strong, blockquote
  • hr

Layout Containers

.d-container

A flexbox container with column direction to wrap sections you wish to center within the viewport and limit the width (default 120ch).

.d-column-container

If placed within .d-container there will be up to four columns on large viewports. If not contained, there will be as many as the viewport allows.

.d-section

Given the body is flex layout, enforces a full-width section for when you want to apply a different background color, for instance. Also clears bottom margin from the last child.

Use the .d-section--padded modifier to add vertical space (default 10vh).

Layout Utilities

Available utilties primarily various spacing modifications.

.d-content-maxlength

While the article element is inherently limited, selectively apply this limitation to other elements (default 80ch).

Margin Spacing Classes

A select set of margin classes are available using "t-shirt sizing" on the following default scale which can be overridden:

          
$d-spacing: (
  "sm": 1rem,
  "md": 3rem,
  "lg": 6rem,
  "xl": 10rem,
  "auto": "auto",
);

Available margin class structures:

  • .d-mx-auto - auto horizontal margin
  • .d-mt-[size] - top margin using value of size key
  • .d-mb-none - remove margin bottom

Margin classes have been selectively created to be general use case without cluttering and bloating the final stylesheet. If you wish to alter them, they are generated in the utilities file.
